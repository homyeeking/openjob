How It Works

Three steps to autonomous AI-powered automation

01

Create a Job

Add a folder with a JOB.md file. Define the cron schedule, conditions, and instructions in Markdown.

02

Schedule Triggers

The scheduler checks your cron expression and evaluates conditions. Jobs only run when the time is right.

03

AI Executes

Claude reads your instructions, composes Skills, and executes the task autonomously. Results are logged and committed.

Jobs vs Skills

Understanding when to use each

Aspect Skills (In Loop) Jobs (Out of Loop)
Trigger Agent decision / Manual /slash Cron schedule + Condition check
Lifecycle Bound to current Agent Loop System-level, persists across sessions
Automation Semi-automated Fully automated
Use Case Interactive, on-demand tasks Recurring, scheduled, background tasks

Specification

Everything you need to know to create your own Jobs

{ } Frontmatter

YAML frontmatter defines metadata: name, cron, description, condition, tags, timeout, retry, and allowedSkills.

* Cron Expressions

Standard 5-field cron format. Supports *, ,, -, and /step operators for flexible scheduling.

? Conditions

Optional execution gates. Use a script path (exit code 0 = proceed) or natural language for AI-evaluated conditions.

# Markdown Body

After frontmatter, write instructions in Markdown. Define objectives, steps, output requirements, and safety notes for the AI.

→ Execution Flow

Schedule check → Condition evaluation &rash; Context loading → Task execution → Result feedback with structured logging.

◊ Best Practices

Keep jobs idempotent, single-purpose, and under 30 minutes. Use environment variables for secrets. Track execution state.
