# Claude 更新收集

## 目标

每天检查 Claude 相关的官方更新，如果有新内容则进行总结并发布到个人博客。

## 执行步骤

### 阶段 1: 信息收集

检查以下来源是否有新内容：

1. **Anthropic 官方博客**
   - https://www.anthropic.com/news
   - 查找最近 24 小时内的新文章

2. **Claude 文档更新**
   - https://docs.anthropic.com
   - 检查变更日志、发布说明

3. **GitHub 发布**
   - 相关 SDK 和工具的更新

### 阶段 2: 内容总结

如果发现新内容：

1. 阅读并理解更新内容
2. 提取关键点：
   - 新功能
   - 重要改进
   - 破坏性变更
   - 新的 API 或参数

3. 撰写一篇结构化的总结文章，包含：
   - 更新概述
   - 核心变化详解
   - 实际应用场景
   - 迁移建议（如有破坏性变更）

### 阶段 3: 发布

使用 `write-to-blog` skill 将总结文章发布到个人博客。

## 输出要求

- 如果有更新：发布一篇博客文章
- 如果无更新：记录"今日无更新"并结束
- 所有执行情况都要记录日志

## 注意事项

- 只处理真正的"新"内容，避免重复发布
- 总结要准确传达官方信息，不要添加臆测
- 如果更新涉及安全问题，要特别强调
- 包含指向官方来源的链接

## 判定条件示例

- 官方博客有新文章发布 → 触发
- 文档有重要更新 → 触发
- 只有小的错别字修复 → 可跳过
